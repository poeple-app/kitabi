Kitabi ne yapıyor?

Kitabi, Telegram'da yaşayan kişisel bir okuma günlüğü botudur: okurken sesli mesaj, sayfa fotoğrafı veya yazılı not gönderirsin, bot bunları Gemini AI ile metne döker, kategorilere ayırır ve okuduğun kitaba bağlayarak SQLite veritabanına kaydeder; kitap bittiğinde de istatistikleri, alıntıları ve favorilerinle birlikte tasarımlı bir PDF okuma günlüğüne dönüştürür.

Manuel Kurulum

Tek sayfada Kitabi kurulumu

Yazılım bilen kullanıcılar için kompakt referans. Sihirbazın 9 adımının tamamı burada, sadece olması gerekenler ve kabuk komutlarıyla. Tahmini süre: 15–25 dk.

Önkoşullar

  • Telegram hesabı (mobil ya da web)
  • Google hesabı (faturalama bağlı olmasına gerek yok — free tier yeterli)
  • GitHub hesabı
  • gcloud CLI — bu sayfadaki komutların tamamı buna dayalı. Lokal kurulum istemiyorsan Cloud Shell'i aç, gcloud orada hazır + senin hesabınla authenticated gelir.

1 · Telegram bot + kendi user ID

  • @BotFather → /newbot → token al
  • @userinfobot → /start → kullanıcı ID al (sadece rakam)

2 · Gemini API key

  • aistudio.google.com/app/apikey → Create API key → kopyala
  • Free tier: gemini-2.5-flash 10 RPM, 250 RPD

3 · Kaynak kodu al

İki yol: (A) GitHub fork — güncelleme almak için ideal. (B) ZIP indir — GitHub hesabı bile gerekmez, tek seferlik kurulum.

A) GitHub fork (güncelleme almak istersen)

  • github.com/poeple-app/kitabi/fork → fork
  • Cloud Shell'de: git clone <fork_url> && cd kitabi

B) ZIP indir (GitHub hesabı yok)

Cloud Shell'i aç ve şu tek satırı çalıştır — ZIP'i indirir, açar, içine girer:

curl -L https://github.com/poeple-app/kitabi/archive/refs/heads/main.tar.gz | tar xz cd kitabi-main

Güncelleme almak istediğinde aynı komutu yeniden çalıştırırsın — eski klasörü silip yeniden indirir.

4 · GCP project + API'ler

  • Project yarat (örn kitabi-prod) — billing zorunlu değil, sadece free tier kullanıyoruz
  • 4 API enable: secretmanager, run, storage, artifactregistry .googleapis.com
# gcloud ile: gcloud config set project <PROJECT_ID> gcloud services enable secretmanager.googleapis.com run.googleapis.com \ storage.googleapis.com artifactregistry.googleapis.com

5 · Cloud Storage bucket (kalıcı SQLite)

  • Region: Cloud Run ile aynı (örn europe-west1)
  • Uniform access, public access prevention: Enforced
gcloud storage buckets create gs://<BUCKET_NAME> \ --location=europe-west1 \ --uniform-bucket-level-access \ --public-access-prevention

6 · Secret Manager — 4 secret

# webhook-secret üret (32 byte hex) WH=$(openssl rand -hex 32) printf "%s" "<TELEGRAM_TOKEN>" | gcloud secrets create telegram-bot-token --data-file=- printf "%s" "$WH" | gcloud secrets create webhook-secret --data-file=- printf "%s" "<YOUR_TG_USER_ID>" | gcloud secrets create allowed-tg-user-ids --data-file=- printf "%s" "<GEMINI_API_KEY>" | gcloud secrets create gemini-api-key --data-file=-

7 · Deploy

gcloud run deploy kitabi \ --source . \ --region=europe-west1 \ --allow-unauthenticated \ --max-instances=1 \ --memory=512Mi \ --set-env-vars="GCS_BUCKET_NAME=<BUCKET_NAME>,DB_PATH=/data/kitabi.db,LOG_LEVEL=INFO" \ --update-secrets="TELEGRAM_BOT_TOKEN=telegram-bot-token:latest,WEBHOOK_SECRET=webhook-secret:latest,ALLOWED_TG_USER_IDS=allowed-tg-user-ids:latest,GEMINI_API_KEY=gemini-api-key:latest"

Service account'a bucket için roles/storage.objectAdmin gerek; Cloud Run otomatik atamadıysa elle ver.

8 · BOT_BASE_URL ve redeploy

URL=$(gcloud run services describe kitabi --region=europe-west1 --format='value(status.url)') gcloud run services update kitabi --region=europe-west1 \ --update-env-vars="BOT_BASE_URL=$URL"

Yeni revision açılınca bot kendi setWebhook'unu çağırır. Doğrulama için Cloud Run loglarında main.webhook.registered event'ini ara.

9 · Cloud Scheduler (opsiyonel — proaktif hatırlatma)

WH=$(gcloud secrets versions access latest --secret=webhook-secret) gcloud scheduler jobs create http kitabi-daily-nudge \ --location=europe-west1 \ --schedule="0 9 * * *" \ --time-zone="Europe/Istanbul" \ --uri="$URL/cron/nudge" \ --http-method=POST \ --headers="X-Telegram-Bot-Api-Secret-Token=$WH"

Atlanabilir. Bot ayar ekranından da kapatılabilir (nudge_enabled).

Mimari özet

  • Stack: FastAPI + python-telegram-bot 21+ webhook mode, SQLAlchemy 2 + SQLite (WAL), google-genai (Gemini), WeasyPrint, structlog JSON
  • Modüller: kitabi/main.py (FastAPI app + lifespan), bot.py (handlers/screens), data.py (ORM + GCS backup + FTS5 + export), ai.py (Gemini wrapper, 3-model fallback)
  • Persistence: Container disk ephemeral; data.periodic_sync_loop her 60s SQLite Online Backup API ile bucket'a snapshot, shutdown'da force-flush
  • Auth: Webhook X-Telegram-Bot-Api-Secret-Token header'ı hmac.compare_digest ile doğrulanır; sonra ALLOWED_TG_USER_IDS allowlist
  • State: Chat-level ephemeral state SQLite'ta ephemeral_state tablosunda; cold-start'ta hayatta kalır

Adım 5 — Cloud Storage Bucket
⏱ ~2 dakika 🗄️ Veritabanı yedeği 🆓 5 GB ücretsiz ✋ Manuel

Botun veritabanı için bir kutu yarat

Cloud Run'ın diski kalıcı değil — container yeniden başlayınca her şey silinir. Bu yüzden botun SQLite veritabanını her 60 saniyede bir bu "kutu"ya (Cloud Storage bucket) yedekleyeceğiz, container açılırken de geri yükleyeceğiz. Bu olmadan kitap eklediğinde 5-10 dakika sonra hepsi gider.

Bucket bir dosya kutusu gibidir — sınırsız klasör, dosya tutabilir. Biz sadece bir tane SQLite dosyası (kitabi.db) tutacağız. 5 GB'a kadar ücretsiz; senin veritabanın ömründe muhtemelen 10 MB'ı geçmez.

Bucket adı global olarak benzersiz olmalı (tüm Google Cloud'da). kitabi-<senin-github-kullanıcı-adın> öner.

Adım 6 — Google Cloud Secret Manager
⏱ ~5 dakika 🔒 Şifreli kasa 🆓 Ücretsiz ✋ Manuel onay

Tüm anahtarları güvenli kasaya koy

Şimdiye kadar topladığın token ve key'leri (Telegram bot token, Telegram user ID, Gemini API key) Google Cloud Secret Manager'a koyacaksın. Bu, senin Google Cloud Project'inin içindeki şifreli kasası — kimse okuyamaz, bot çalışırken oradan çekecek.

Neden kasa? Çünkü token'lar şifre gibidir. Bilgisayardaki bir dosyada yazılı dursa, bilgisayar başkasının eline geçince çalınabilir. Secret Manager'da: sadece senin Google hesabın görebilir, şifreli depolanır, değiştirildiğinde versiyonu kaydedilir, bot her seferinde yeniden çeker.

Bu adımda 4 secret oluşturacaksın. Her birinin adı sabit — adlar bu olmazsa bot çekemez.

🔒 Bu adımda kurulum hiçbir secret görmüyor. Sen doğrudan Google'ın console'una yapıştırıyorsun. Kurulum sadece "şu adla secret oluştur, şu değeri koy" diyor.

Oluşturulacak secret'lar (4 adet)

webhook-secret nedir, niye rastgele?

Bot Cloud Run'da public bir URL'de yaşıyor. Birisi bu URL'i keşfederse, sahte mesaj atıp botu kandırabilir. Önlem: Telegram her webhook isteğine secret bir token ekliyor. Bot bu token'ı kontrol ediyor — eşleşmezse isteği çöpe atıyor. Token'ın tahmin edilemez olması için browser'ın kriptografik rastgele üreteciyle 256-bit (64 karakter hex) üretiyoruz. Brute-force pratik olarak imkânsız.

webhook-secret üreteci

Browser'ın crypto.getRandomValues fonksiyonu üretiyor. Dışarı çıkmıyor — sayfayı kapatınca kaybolur.

Adım 7 — Deploy
⏱ ~5 dakika (build dahil) 🌐 deploy.cloud.run 🚀 Tek tık ✅ Otomatik doğrulama

Şimdi botu Google Cloud Run'a kur

Şimdi kodu Google Cloud Run'a "kuruyoruz" (deploy). Google'ın resmi "Deploy to Cloud Run" sayfası kodu GitHub'daki repondan otomatik çekip Cloud Run'a yükleyecek.

İlk deploy 3-5 dakika sürer çünkü Google önce kodu build ediyor (Python paketlerini yüklüyor, Docker image üretiyor). Sonraki güncellemeler 1 dakikaya iner.

Deploy ekranında dikkat: 4 secret'ı "Reference a secret" ile bağlamayı ve GCS_BUCKET_NAME plain env'ini girmeyi unutma. Yoksa bot başlarken "token boş" der ya da veri kalıcı olmaz.

Adım 8 — Bağlantı (BOT_BASE_URL + Webhook)
⏱ ~2 dakika 🔗 Telegram ↔ Cloud Run ✋ Manuel onay

Son adım: Telegram'a botun adresini söyle

Şu an bot ayakta ama Telegram nereye mesaj göndereceğini bilmiyor — çünkü Adım 7'de bot'u kurarken Cloud Run sana hangi adresi vereceğini henüz bilmiyorduk, oradaki adres kutusunu boş bırakmıştık. Şimdi o adresi bot'a yazacağız. Bot bunu öğrenir öğrenmez Telegram'a "ben buradayım, mesajları buraya yolla" diyecek.

Teknik adı: webhook. "Telegram, kullanıcı mesaj atınca şu URL'e ilet" demek. webhook-secret de sahte istekleri kapıda döndüren bir parola gibi.

1) Cloud Run'a bot'un kendi adresini söyle

Bunun için bot'u yeniden başlatmamız gerek — yeni bir bilgiyle açılsın diye. Cloud Run buna "yeni revision" diyor; tek tıkla oluyor, ~1 dakika sürer.

  1. Cloud Run console'da kitabi servisini aç
  2. Üstteki EDIT & DEPLOY NEW REVISION butonuna bas
  3. Variables & Secrets sekmesine geç
  4. Listede BOT_BASE_URL satırını bul (Adım 7'de boş bırakmıştın)
  5. Değer kutusuna Cloud Run'ın sana verdiği URL'i yapıştır
  6. Alt sağdaki DEPLOY butonuna bas

Bot bu adresi öğrenir öğrenmez kendi kendine Telegram'a haber veriyor. Yani ek bir şey yapmana gerek yok — sadece bağlantının kurulduğunu kontrol et.

2) Bağlantı kuruldu mu — kontrol

İki yol var: (a) Cloud Run loglarında "webhook hazır" mesajını ara (kolayı bu). (b) Cloud Shell'den aşağıdaki komutla manuel kontrol et.

Yöntem A: Cloud Shell (token'a dokunmadan)

Cloud Shell'i aç
https://shell.cloud.google.com
Google Cloud'un yerleşik terminali, gcloud zaten authenticated
↗

Açılan terminale aşağıdakini yapıştır (Cloud Run URL'ini sen koyduğun için zaten dolu):

curl 'https://api.telegram.org/bot$(gcloud secrets versions access latest --secret=telegram-bot-token)/setWebhook?url=<CloudRunURL>&secret_token=$(gcloud secrets versions access latest --secret=webhook-secret)'

Cloud Shell gcloud secrets versions access ile Secret Manager'dan token'ı çekiyor — değerler ekranda görünmüyor, sadece curl ile yollanıyor.

Yöntem B: Bot kendi kendine bağlasın (önerilen)

Yukarıdaki adımdan sonra bot zaten kendiliğinden Telegram'a "ben buradayım" diyor. Doğrulamak için Cloud Run loglarını aç ve "webhook hazır" ya da "main.webhook.registered" yazan bir satır ara. Bu satırı görürsen bağlantı kurulmuş demektir.

Cloud Run loglarını gör
https://console.cloud.google.com/logs
"webhook hazır" mesajını ara

Adım 9 — Cloud Scheduler (opsiyonel)
⏱ ~3 dakika 🔔 Hatırlatma için 🆓 3 job/ay ücretsiz ✋ Opsiyonel

Hatırlatma için Cloud Scheduler (atlanabilir)

Bot, "uzun süredir okumadığın kitap" varsa sana otomatik mesaj atabiliyor. Bunun için Google Cloud Scheduler'a günde bir kez botun /cron/nudge endpoint'ini tetikleten bir iş kurman gerek. Atlarsan bot yine çalışır — sadece proaktif hatırlatma olmaz; Telegram'dan kendin gittiğinde her şey aynı.

Cloud Scheduler ücretsiz katmanda ayda 3 job veriyor; biz 1 tane kullanacağız.

Cloud Scheduler'ı aç
https://console.cloud.google.com/cloudscheduler
Project ID girersen direkt project'inin Scheduler'ı açılır
↗

Job ayarları

Header değerini nasıl alacağım? Adım 6'da webhook-secret'ı üretmiştin ve Secret Manager'a koymuştun. Cloud Shell'de aşağıdaki komutla göstertebilirsin — sonra Scheduler header'ına yapıştır:
gcloud secrets versions access latest --secret=webhook-secret
Ayar değiştirmek istersen: Bot içinden Ana Menü → Ayarlar → "Hatırlatmayı kapat" / "± gün" ile yönetirsin. Scheduler her gün tetikler ama bot ayarına göre mesaj atıp atmadığına karar verir. Yani tamamen kapatmak için job'ı silmen gerekmez; bot ayarından kapat.
